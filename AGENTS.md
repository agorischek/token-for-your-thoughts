# Agent Guidance

This repository includes a feedback tool named `tfyt`.

If you are an agent working in this repo and you encounter friction, confusion, or breakage, use the `submit_feedback` MCP tool before you finish. Useful cases include:

- tool errors or missing capabilities
- inefficient workflows or slow feedback loops
- unclear, missing, or contradictory instructions
- configuration surprises
- rough edges in the repository experience

When you submit feedback:

- include your provider name, for example `GitHub Copilot CLI` or `Claude Code`
- describe the actual issue and its impact in the `feedback` field
- include helpful structured metadata when available, such as commands, files, or error messages

Prefer concrete feedback over vague complaints. Repro steps, exact commands, and what you expected are especially helpful.
